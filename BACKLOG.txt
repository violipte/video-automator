========================== DONE (CONCLUIDO) ====================================

[x] VIDEO EDITING ENGINE (engine.py)
    - OpenCV zoom com ease-in-out cubico (warpAffine + subpixel interpolation)
    - 4 efeitos Ken Burns: zoom_in, zoom_out, pan_left, pan_right (alternados)
    - Cache de clips por hash MD5 (img path + dur + resolucao + fps + zoom + efeito)
    - FFmpeg concat de clips pre-renderizados + overlay + legendas + audio
    - Anti-jitter: coordenadas float no OpenCV, interpolacao INTER_LINEAR
    - Imagens escaladas via INTER_LANCZOS4 para tamanho de trabalho
    - Frames enviados por pipe rawvideo ao FFmpeg para encode H.264
    - Fallback automatico h264_nvenc -> libx264 para clips
    - Renderizacao final com h264_nvenc, CRF 20, AAC 192k, faststart
    - Ajustes visuais Lumetri-style: exposicao (gamma), brilho, contraste,
      saturacao, realces, sombras, brancos, pretos, temperatura, tonalidade, vinheta
    - Randomizacao de ajustes visuais com variacoes sutis centradas nos valores base
    - Filtros FFmpeg: eq, curves, colorbalance, vignette
    - Tipo fundo: imagens (com zoom) ou videos (sem zoom, com scale+pad+fps)
    - Suporte a multiplos overlays por template com opacidade individual
    - Filter complex escrito em arquivo temporario para evitar limite de cmd line
    - Callback de progresso por etapa (clips 0-60%, render 65-100%)
    - Cancelamento mid-render com processo.kill()

[x] FFMPEG PRIORITY MANAGEMENT
    - Processos FFmpeg rodam com BELOW_NORMAL_PRIORITY_CLASS via psutil
    - Fallback sem psutil via ctypes/kernel32 para Windows
    - Evita travar o servidor durante renders pesados

[x] SUBTITLE SYSTEM (subtitle_fixer.py + engine.py)
    - ASS format com PlayResX/Y = resolucao real do video (pixel-perfect)
    - Whisper transcricao (faster-whisper com GPU CUDA, fallback CPU)
    - Correcao automatica de SRT: substituicoes, hesitacoes, capitalizacao
    - Regras por idioma (en/de/pt/es) em rules/*.json com merge global + template
    - Regras embutidas no template (prioridade sobre arquivo JSON)
    - Quebra de linha inteligente: equilibrio entre linhas, nunca corta palavras
    - Remocao de hesitacoes por idioma (uh, um, ah, eh, tipo, naja, etc)
    - Palavras customizadas para remover
    - SRT -> ASS conversion com estilo completo (fonte, tamanho, cor, outline, sombra)
    - Font scaling formula: FontSize * (resolucao_real / 288)
    - Outline e Shadow tambem escalados pela mesma formula
    - 5 estilos preset + customizacao completa via legenda_config
    - Posicionamento: bottom, top, center, custom (X%/Y%)
    - Maiuscula opcional por template
    - Max chars por linha e max linhas configuraveis
    - Liberacao de VRAM apos transcricao (del model + gc + torch.cuda.empty_cache)

    [ ] Salvar roteiro gerado na celula

    ETAPA 2 - NARRAR
    [ ] Pegar o roteiro gerado na etapa anterior
    [ ] Usar a voz configurada no template associado à coluna
    [ ] Gerar TTS via ai33.pro (ElevenLabs ou Minimax)

    [ ] Salvar em pasta configurada como TAG DD-MM.mp3

    [ ] Rodar Whisper no MP3 gerado
    [ ] Aplicar regras do template (substituicoes, max_chars, maiuscula)
    [ ] Gerar SRT corrigido -> ASS com estilo do template

    ETAPA 4 - PRODUZIR VIDEO
    [ ] Montar video com engine.py usando template da coluna
    [ ] Input: MP3 (narracao) + imagens + overlay + trilha + legenda ASS
    [ ] Efeitos: zoom OpenCV, ajustes visuais, CTA chroma key
    [ ] Output: TAG_YYYYMMDD_SEQ.mp4 na pasta de saida do template

    ETAPA 5 - POS-PRODUCAO (futuro)

[ ] OPÇÃO 1: CLAUDE CODE SCHEDULED AGENT
    Usar o recurso de schedule/cron do Claude Code para criar um agente
    que roda em intervalos programados.
    Tarefas:
    - A cada 30 min: verificar se servidor está rodando
    - Checar se há produções travadas ou com erro
    - Verificar se há temas pendentes para a data de hoje
    - Relatar status via log ou webhook
    - Reiniciar servidor se caiu
    Vantagem: integrado com Claude Code, fácil de configurar
    Limitação: roda em intervalos, não é contínuo
    Prioridade: MEDIA

[ ] OPÇÃO 2: AGENT SDK (AGENTE INTELIGENTE)
    Usar claude_agent_sdk da Anthropic para criar agente autônomo
    que roda continuamente com acesso às APIs do app.
    Tarefas:
    - Monitorar status de produções em tempo real
    - Detectar falhas e re-executar automaticamente
    - Verificar temas pendentes e iniciar produção proativamente
    - Tomar decisões: "roteiro muito curto → re-gerar"
    - Validar qualidade: contar chars, verificar estrutura do roteiro
    - Reportar via webhook (Telegram, Discord)
    - Sugerir temas baseado em tendências anteriores
    Vantagem: mais inteligente, toma decisões, contínuo
    Limitação: consome créditos de API, mais complexo
    Prioridade: BAIXA

[ ] OPÇÃO 3: MONITOR PYTHON (PRÁTICO / SEM API)
    Script monitor.py que roda em loop independente do app.
    Tarefas:
    - Loop infinito com checagem a cada N minutos
    - Verificar saúde: GET /api/batch/status
    - Se servidor morto → reiniciar (subprocess)
    - Se produção travada (>30 min sem progresso) → cancelar e retentar
    - Se hora configurada (ex: 06:00) → iniciar produção da data de hoje
    - Se hora configurada (ex: 22:00) → gerar relatório do dia
    - Log tudo em arquivo rotativo
    - Webhook opcional para notificações
    Vantagem: robusta, sem custo de API, roda 24/7
    Limitação: não toma decisões inteligentes, só regras fixas
    Prioridade: ALTA

    ```
    monitor.py
    - check_server_health() → ping /api/batch/status
    - check_stuck_production() → se ativo>30min, cancelar
    - auto_start_production(data) → POST /api/producao-completa
    - restart_server() → subprocess python app.py
    - send_notification(msg) → webhook POST
    - main loop: while True → check → sleep(interval)
    ```
